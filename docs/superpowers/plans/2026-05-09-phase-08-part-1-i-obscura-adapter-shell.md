## Why this scope (research summary)

Obscura (`https://github.com/h4ckf0r0day/obscura`, Apache 2.0, Rust) ships in two modes:

1. **Stateless one-shot CLI** — `obscura fetch <url>` (single page) + `obscura scrape <urls...>` (parallel multi-URL).
2. **CDP server daemon** — `obscura serve --port 9222` (long-running; Puppeteer/Playwright connect via `connectOverCDP`).

Mode 2 overlaps with the existing `playwright-lib` adapter's transport (a future `--cdp-endpoint` flag on playwright-lib reaches it). **Mode 1 is the unique lane** — stateless `--scrape` (no upstream peer), `--stealth` (3520-tracker block + anti-fingerprinting), 30 MB / 70 MB binary.

**Decision:** Phase 8 opens stateless. Daemon-lifecycle verbs not added; `obscura serve` stays out-of-scope (covered by playwright-lib later if demand surfaces). 8-1-i ships the adapter shell only — no real verb backend. 8-1-ii fills `tool_extract --scrape`. 8-1-iii fills `tool_extract --stealth`.

Path A "ship-without-promotion" per adapter-extension-model spec §4.4: adapter is reachable via `--tool=obscura` but **not** referenced from `router.sh`. Zero precedence-rule risk in this PR; promotion lives in a future 8-2-i PR.

## API surface

### `scripts/lib/tool/obscura.sh`

Standard adapter ABI (per spec 2026-04-30 §2):

```bash
tool_metadata           # { name: "obscura", abi_version: 1, version_pin: "0.x", cheatsheet_path }
tool_capabilities       # { verbs: { extract: { flags: [...] } } }
tool_doctor_check       # { ok, binary, version | error, install_hint }

tool_open               # return 41 (UNSUPPORTED_OP) — no navigation (one-shot fetch only)
tool_click              # return 41
tool_fill               # return 41
tool_snapshot           # return 41
tool_inspect            # return 41
tool_audit              # return 41
tool_extract            # return 41 — STUB; real impl in 8-1-ii (scrape) / 8-1-iii (stealth)
tool_eval               # return 41
```

Identity-only declarations in `tool_capabilities`: `extract` is the only verb. Every other verb-dispatch function is a 41-stub. Defensive (router would not route here without `--tool=obscura`); cheap.

### `references/obscura-cheatsheet.md`

New cheatsheet doc — mirrors `playwright-cli-cheatsheet.md` shape:
- When the router picks this adapter (`--scrape` / `--stealth`; not yet promoted to default — Path B in 8-2-i)
- Capabilities declared (the JSON above)
- Doctor check — binary discovery + install hint
- Version pin — `0.x` (pre-1.0 upstream)
- Override — `--tool=obscura` Path A entry
- Limitations — no navigation, no console/network HAR (use playwright-cli / chrome-devtools-mcp)

### `references/tool-versions.md` + `SKILL.md` tools-table

Both autogenerated by `scripts/regenerate-docs.sh all`. The new adapter row appears in both.
