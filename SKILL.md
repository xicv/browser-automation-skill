# browser-automation-skill (Phase 2 ‚Äî site & session core)

Phase 2 ships site CRUD + the session schema + a stub-adapter `login`.
Real browser launches arrive in Phase 3.

## Verbs

| Verb | What it does | Example |
|---|---|---|
| `doctor`        | Health check: deps, state dir mode, disk encryption, no network | `bash "${CLAUDE_SKILL_DIR}/scripts/browser-doctor.sh"` |
| `add-site`      | Register a site profile | `‚Ä¶ add-site --name prod --url https://app.example.com` |
| `list-sites`    | List registered sites | `‚Ä¶ list-sites` |
| `show-site`     | Show one site's profile JSON | `‚Ä¶ show-site --name prod` |
| `remove-site`   | Typed-name confirmed delete | `‚Ä¶ remove-site --name prod --yes-i-know` |
| `use`           | Get / set / clear current site | `‚Ä¶ use --set prod` |
| `login`         | Capture a Playwright storageState into a session | `‚Ä¶ login --site prod --as prod--admin --interactive` |
| `login` (file)  | Same ‚Äî but consume a hand-edited storageState file | `‚Ä¶ login --site prod --as prod--admin --storage-state-file PATH` |
| `list-sessions` | List captured sessions (optionally filter by site) | `‚Ä¶ list-sessions --site prod` |
| `show-session`  | Show session metadata (NEVER cookie/token values) | `‚Ä¶ show-session --as prod--admin` |
| `remove-session`| Typed-name confirmed delete of a captured session | `‚Ä¶ remove-session --as prod--admin --yes-i-know` |
| `creds add`     | Register credential (smart per-OS backend; AP-7 stdin-only) | `printf 'pw' \| ‚Ä¶ creds-add --site prod --as prod--admin --password-stdin` |
| `creds list`    | List credentials (optional `--site` filter; metadata only) | `‚Ä¶ creds-list --site prod` |
| `creds show`    | Show credential metadata (NEVER secret value) | `‚Ä¶ creds-show --as prod--admin` |
| `creds remove`  | Typed-name confirmed delete of a credential | `‚Ä¶ creds-remove --as prod--admin --yes-i-know` |
| `open`          | Open a URL in the picked browser adapter | `‚Ä¶ open --url https://app.example.com` |
| `open` w/ session | Apply a stored storageState before navigating | `‚Ä¶ open --site prod --as prod--admin --url ‚Ä¶` |
| `snapshot`      | Capture an `eN`-indexed accessibility snapshot | `‚Ä¶ snapshot` |
| `click`         | Click an element by `--ref eN` or `--selector CSS` | `‚Ä¶ click --ref e3` |
| `fill`          | Fill an input ‚Äî `--text VALUE` or `--secret-stdin` | `‚Ä¶ fill --ref e3 --text "search query"` |
| `inspect`       | Inspect an element by `--selector CSS` | `‚Ä¶ inspect --selector "h1"` |

`${CLAUDE_SKILL_DIR}` is the absolute path that Claude Code injects when it
invokes the skill ‚Äî it points at the symlink under `~/.claude/skills/`. Use it
in command examples so they work whether the user installed at `--user` or
`--project` scope.

## Before running anything

If `doctor` reports `~/.browser-skill` missing, run `./install.sh` (or
`./install.sh --with-hooks` for the credential-leak blocker).

## Output contract

Every verb prints zero or more streaming JSON lines, then ends with a
single-line JSON summary. Parse with jq; route on `.status` (`ok`,
`partial`, `error`, `empty`, `aborted`).

```
$ bash scripts/browser-doctor.sh | tail -1 | jq .
{"verb":"doctor","tool":"none","why":"health-check","status":"ok","problems":0,"duration_ms":42}
```
